# ADR-0005: Add PR continuous integration

- Status: Accepted
- Date: 2026-07-14

## Context

The only workflow is `.github/workflows/beta-release.yml`, which builds and
publishes on `v*-beta*` tags. Nothing validates pull requests, so lint/type/test
regressions reach `main` freely. Coverage thresholds (80/75) exist but are not
enforced.

## Decision

Add `.github/workflows/ci.yml` triggered on `pull_request` and `push` to `main`.
Steps: install → `npm run lint` (`--max-warnings=0`) → `npm run type-check` →
`npm run test:run` → `npm run test:coverage` (thresholds enforced once component
tests land in Phase 4). Cache npm. Fail the check on any step failure.

## Consequences

- Regressions are caught before merge.
- Coverage becomes a real gate rather than aspirational config.
- Release workflow is unchanged and remains tag-triggered.
