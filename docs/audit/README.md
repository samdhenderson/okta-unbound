# Audit reports

Point-in-time maintainability audits of the codebase. Each report scores the
code against the specs in `docs/` and produces a remediation backlog tagged
**quick-win** vs **large-refactor**.

| Report                                 | Date       | Summary                                                 |
| -------------------------------------- | ---------- | ------------------------------------------------------- |
| [2026-07-audit.md](./2026-07-audit.md) | 2026-07-14 | Baseline audit that seeds the specs + remediation plan. |

Re-run the audit after each major remediation wave and add a new dated report;
do not edit old ones (they are historical baselines).
