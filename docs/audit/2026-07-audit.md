# Baseline maintainability audit — 2026-07-14

Scope: full `src/` (~22k LOC, 109 files), tooling, and Claude-context surface.
Each finding cites `file:line` evidence and is tagged **[QW]** quick-win or
**[LR]** large-refactor. Severity: 🔴 high / 🟠 medium / 🟡 low.

## Summary

The extension is functionally rich and has a genuinely good foundation — a
message-passing pipeline with rate limiting, a decomposed `useOktaApi/`, a design
token system, and a shared component library. The gaps are consistency and
guardrails: tokens/components are bypassed in places, there's no runtime
validation or logging discipline, a few god components concentrate risk, and
quality gates (Prettier, strict ESLint, PR CI, component tests) are missing.

| Area            | Grade | Headline                                                   |
| --------------- | ----- | ---------------------------------------------------------- |
| Design system   | C     | Good tokens, bypassed by raw hex                           |
| Components      | C+    | Good library, incomplete barrel, 54 raw `<button>`s        |
| UX / a11y       | C     | Modal has no dialog semantics / focus trap / Escape        |
| Architecture    | B     | Clean pipeline; two god components + a mega content script |
| Type safety     | C     | 68 `any`s, no runtime validation of Okta responses         |

| Tooling/CI      | D     | No Prettier, lax ESLint, no PR CI                          |

## 1. Design system 🟠

- **[QW]** Raw hex bypassing Odyssey tokens:
  - `SchedulerStatusBar.tsx:42-52` — `#22c55e`, `#2196f3`, `#ff9800`, `#f44336`,
    `#9e9e9e` for status colors; these don't even match `--color-success` (#16884a) etc.
  - `ContextBanner.tsx:102-108` — purple `#9333ea`/`#7e22ce` (not in the token set)
    and a mix of `var(--color-success)` with hardcoded `#127a40` in one object.
  - `overview/members/AttributeFacet.tsx:19-21` — a 6-stop dataviz `RAMP` + neutral
    hexes. Legitimate need (sequential ramp), but should be tokenized/documented as
    a chart palette, not inline literals.
- Tokens themselves are well-formed in `src/sidepanel/tailwind.css` (`@theme`,
  lines 1-35). Keep them as the single source; forbid raw hex outside this file.

## 2. Components 🟠

- **[QW]** Barrel `components/shared/index.ts` exports only 7 of 13 components
  (missing `AlertMessage`, `EmptyState`, `LoadingSpinner`, `ScrollableList`,
  `SearchDropdown`, `SelectionChips`) → consumers deep-import inconsistently.
- **[QW]** 54 raw `<button>` elements in feature components bypass shared `Button`
  (e.g. `GroupsTab.tsx:514,581,608,632` with ad-hoc `px-2.5 py-1.5` chips vs the
  `sm|md|lg` scale). Also `RulesTab.tsx`, `GroupCollections.tsx`, `SearchDropdown.tsx`.
- **[QW]** Status vocabulary split: `AlertMessage` uses `'error'`, `Button` uses
  `'danger'`; 29 `'error'` vs 8 `'danger'` repo-wide. No shared `StatusType`. → ADR-0002.
- **[QW]** `Button.tsx:58-62,96` hardcodes pixel heights (`36/40/56px`) via inline
  `style` duplicating the Tailwind `sizeClasses` — collapse into one system.
- Established good convention to standardize on: the `Record<Variant,string>`
  lookup-map used by `Button`, `Icon`, `Modal`, `AlertMessage`.

## 3. UX / accessibility 🔴

- **[QW]** `components/shared/Modal.tsx` has **no** `role="dialog"`, `aria-modal`,
  focus trap, autofocus, focus restore, or Escape-to-close (closes only on overlay
  click, line ~32). This is the single overlay used by all 8 modal features.
- **[LR]** `UserComparisonModal.tsx` tablist (lines ~577-586) has `role`/`aria-selected`
  but lacks `aria-controls`/`tabpanel` wiring and roving `tabIndex`/arrow keys.
- Positive: `Button` has a visible focus ring; icons use `aria-hidden` correctly.

## 4. Architecture 🟠

- Clean, documented pipeline: Sidepanel `useOktaApi` → Background `ApiScheduler`
  (rate limit, `shared/scheduler/apiScheduler.ts`, 477 lines) → Content
  `content/index.ts` fetch → Okta. `useOktaApi/` is well decomposed (~13 modules).
- **[QW]** Two paths to the content script — some ops go sidepanel→content directly
  (`useOktaApi/core.ts:31`), others via background (`core.ts:50`). The direct path
  **bypasses the rate limiter**. Standardize on the scheduler path.
- **[LR]** God components concentrate risk and block testing:
  `UsersTab.tsx` (1364 lines, 19 `useState`), `GroupsTab.tsx` (935, 23 `useState`),
  `UserComparisonModal.tsx` (967), `content/index.ts` (1344).
- **[LR]** `useGroupContext.ts` and `useUserContext.ts` are ~227 lines each and
  near-identical (tab discovery, retry/backoff, debounce, stale-guard) → extract a
  shared `useOktaTabContext` base.

## 5. Type safety 🟠

- **[LR]** 68 `any`s (non-test). Hotspots: `useOktaApi/core.ts:11,37-41`
  (`body?: any`, `Promise<any>`), `shared/types.ts:189` (`MessageRequest.body`),
  `OktaUser.profile: [key: string]: any` (`types.ts:42`), `ApiResponse<T = any>`.
- **[LR]** No runtime validation of Okta responses — JSON is cast, not parsed. A
  shape change fails silently or crashes at a random deref. → adopt zod (ADR-0006).

## 6. Logging & secrets 🔴

- **[QW/LR]** 324 `console.*` calls (non-test). Heaviest: `content/index.ts` (81),
  `RulesTab.tsx` (23), `useUserContext.ts`/`useGroupContext.ts` (21 each),
  `UsersTab.tsx` (17), `apiScheduler.ts` (16).
- **[QW]** Some leak secrets/payloads: `useOktaApi/core.ts:30-32` (request bodies),
  `content/index.ts:182-184` (XSRF token preview). `no-console` is `off` in ESLint,
  so these ship to production.
- Fix: introduce a level-gated `logger`, forbid raw `console.*` (ADR-0004), and
  never log tokens/payloads.

## 7. Tooling & CI 🟠

- **[QW]** No Prettier — formatting unmanaged. → ADR-0003.
- **[QW]** ESLint flat config downgrades many rules to `warn`/`off`
  (`no-explicit-any`, `no-console`, `no-non-null-assertion`, `set-state-in-effect`,
  `ban-ts-comment`) and hand-declares globals. → ADR-0004.
- **[QW]** Only CI is `beta-release.yml` (tag-triggered). No PR lint/type/test gate.
  → ADR-0005.
- **[QW]** Version drift: `manifest.json` 0.4.0, `package.json` 0.4.0-beta.1,
  `background/index.ts:210,221` hardcodes `0.3.0`. → ADR-0007.
- **[QW]** Dead legacy tab-name migration cruft: `App.tsx:32-46`,
  `background/index.ts:212` reference tabs (`dashboard`, `operations`, `undo`,
  `security`, `apps`) not in the current `TabType`.
- **[QW]** Placeholder metadata: `package.json` `author: "Your Name"`,
  repo `yourusername/okta-unbound`.

## 8. Testing 🔴

- **[LR]** 7 test files, all pure-util unit tests (`ruleEvaluator`, `ruleUtils`,
  `statusNormalizer`, `mfaUtils`, `auditStore`, `memberAnalytics`, `useOktaApi`).
  Zero component/UI tests despite Testing Library + MSW being wired
  (`src/test/mocks/handlers.ts`). Coverage thresholds (80/75) exist but aren't
  enforced (no CI gate).

## 9. Duplication 🟡

- **[QW]** `formatDate`/`getRelativeTime` reimplemented ~3x: `UsersTab.tsx:16-51`,
  `users/UserProfileCard.tsx:55-84`, `shared/utils/csvUtils.ts:8`.
- **[QW]** Okta-domain check (`okta.com`/`oktapreview.com`/`okta-emea.com`) copy-pasted
  ~15 sites (`background/index.ts:315-320`, `useGroupContext.ts` 3×, `useUserContext.ts`,
  `useOktaPageContext.ts`, `ContextBanner.tsx`). No shared `isOktaUrl`.

**Quick wins (Phase 3):** hex→tokens · complete barrel · raw `<button>`→`Button` ·

`formatDate` utils · delete legacy tab cruft · `error`→`danger` codemod · Prettier ·
